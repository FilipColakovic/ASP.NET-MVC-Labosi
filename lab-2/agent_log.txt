```aspnetcorerazor
string GetGroupKey(string key)
    {
        var dotIndex = key.IndexOf('.');
        var bracketIndex = key.IndexOf('[');
        var cutIndex = -1;
        if (dotIndex >= 0 && bracketIndex >= 0)
        {
            cutIndex = Math.Min(dotIndex, bracketIndex);
        }
        else if (dotIndex >= 0)
        {
            cutIndex = dotIndex;
        }
        else if (bracketIndex >= 0)
        {
            cutIndex = bracketIndex;
        }
        if (cutIndex > 0)
        {
            return key.Substring(0, cutIndex);
        }
        return "General";
    }
```


Agent output: This function extracts a **group name** from a property-like key string. It looks for two separator patterns: a dot (`.`), which usually indicates nested properties (for example, `Courier.Name`), and an opening bracket (`[`), which usually indicates indexed values (for example, `Packages[0]`).

It first finds the position of both separators with `IndexOf`. If both exist, it chooses the earliest one with `Math.Min`, because that marks the first boundary of the top-level group. If only one exists, it uses that one. If neither exists, `cutIndex` stays `-1`.

The `cutIndex > 0` check means â€œthere is a separator and it is not the first character.â€ In that case, the function returns everything before that separator using `Substring(0, cutIndex)`. So `Address.City` becomes `Address`, and `Items[2].Name` becomes `Items`.

If no usable separator is found, it returns `"General"` as a fallback bucket. A small gotcha: keys like `.Name` or `[0]Item` will also end up in `"General"` because the separator is at index `0`, and the function intentionally ignores that as an invalid group prefix.
